1 Mekanisme Pengujian

Setiap model proxy ML diuji dengan 1.000 kasus sintetis per penyakit (35 model × 1.000 = 35.000 total prediksi). Test data dihasilkan dengan seeded PRNG (Mulberry32, seed=42) — berarti setiap pengujian akan menghasilkan angka yang identik bit-by-bit. Tidak ada randomness yang tidak terdokumentasi.

1
Generate Test Cases
Mulberry32 PRNG (seed=42) menghasilkan vektor fitur dari rentang fisiologis yang valid (SpO₂: 88–100%, HR: 50–180 bpm, PM2.5: 0–200 µg/m³, dll). Range mengikuti standar physiological monitoring yang umum digunakan.
2
Tentukan Ground Truth (Label Asli)
Label 0/1 ditentukan oleh aturan klinis berdasarkan literatur (SLR) — bukan dari proxy ML itu sendiri. Misalnya: asthma=1 bila PM2.5≥75 µg/m³ DAN SpO₂_deficit≥4%, mengikuti GINA 2024 + WHO PM2.5 protective thresholds.
3
Jalankan Proxy ML
Fitur diteruskan ke MLClient.predictForRisk(riskId, factors) — fungsi prediksi yang sama persis dengan yang dipakai aplikasi di halaman risk-detail.html. Tidak ada modifikasi atau adjustment untuk testing.
4
Bandingkan & Hitung Confusion Matrix
Threshold prediksi default 50% (predicted=1 if risk_pct≥50). Hitung TP/FP/TN/FN untuk masing-masing 35 model.
5
Metrics & Confidence Intervals
Hitung Accuracy, Precision, Recall, F1, Specificity, ROC-AUC (sweep 101 thresholds + trapezoidal integration), dan Expected Calibration Error (ECE, 10-bin). Setiap angka utama disertai Wilson 95% Confidence Interval.
6
Render Laporan Transparan
Hasil di-render ke halaman ini secara real-time. Pengguna dapat klik tiap baris untuk melihat detail per model: confusion matrix, ROC curve, calibration plot, sample data, dan referensi klinis.

2 Formula Matematis (Tanpa Kotak Hitam)

Accuracy
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Precision (Positive Predictive Value)
Precision = TP / (TP + FP)
Recall (Sensitivity)
Recall = TP / (TP + FN)
F1-Score (Harmonic Mean)
F1 = 2 × (Precision × Recall) / (Precision + Recall)
ROC-AUC (Trapezoidal Integration)
AUC = ∑ ((FPRᵢ₊₁ − FPRᵢ) × (TPRᵢ + TPRᵢ₊₁) / 2)
ECE (Expected Calibration Error, 10 bins)
ECE = ∑ (nᵦ / N) × |avg_pred(b) − obs_rate(b)|
Wilson 95% CI (untuk proporsi)
CI = (p̂ + z²/(2n) ± z√(p̂(1−p̂)/n + z²/(4n²))) / (1 + z²/n), z = 1.96

4 Batasan Pengujian (Transparency)

Demi akuntabilitas penuh, perlu disampaikan batasan penting:

1. Data sintetis, bukan klinis nyata. Test menggunakan data yang di-generate dari rentang fisiologis, bukan rekam medis pasien sungguhan. Ini adalah fidelity test (apakah proxy match dengan ekspektasi klinis), bukan clinical validation (apakah proxy match dengan outcome nyata).
2. Ground truth juga deterministik. Aturan klinis disederhanakan menjadi threshold logika. Realitas klinis lebih nuanced (multi-faktor, ko-morbid, individual variability).
3. Calibrated proxy, bukan ML model nyata. Fungsi prediksi adalah weighted sigmoid dengan parameter yang di-kalibrasi terhadap target accuracy dari paper SLR. Model XGBoost/LightGBM/CNN dari paper tidak di-deploy secara native — fungsi proxy meniru behavior dengan cost komputasi <1ms.
4. Untuk validasi klinis nyata, diperlukan: (a) data wearable real-world dari pengguna sungguhan, (b) labeling oleh dokter, (c) IRB approval dan informed consent, (d) IDE/FDA submission jika untuk medical device. AERVINEX saat ini adalah educational/wellness tool, bukan medical device.
5. Reproducibility. Test dapat dijalankan ulang dengan AervinexMLTest.runAll(1000, 42) di console. Seed sama → hasil bit-identical. Jika ada anomali, dapat dilaporkan dengan reproducible test case.

5 Cara Reproduksi & Audit

Setiap orang dapat menjalankan test ini secara independen:

  1. Buka DevTools (F12) di halaman ini
  2. Jalankan: AervinexMLTest.runAll(1000, 42)
  3. Bandingkan output dengan tabel di atas — harus identical
  4. Untuk ubah parameter: AervinexMLTest.runAll(5000, 99) (5000 cases, seed 99)
  5. Source code: public/js/ml-test-runner.js dan public/js/ml-client.js

Tidak ada server-side computation. Semua test berjalan di browser Anda — tidak bisa di-manipulasi oleh kami.
