Encryption at rest = Firebase default (Google-managed keys), belum ada CMEK option.

Action Items

  • P0SCSP + security headers via firebase.json
    Tambah hosting.headers: Content-Security-Policy (script-src self + firebase domains), Strict-Transport-Security, X-Frame-Options: DENY, Referrer-Policy: strict-origin-when-cross-origin, Permissions-Policy.
  • P0MFirestore rules hardening audit
    Jalankan firebase emulators:exec --only firestore + firebase-rules-test. Cek: (a) tidak ada wildcard write, (b) request.auth.uid == resource.data.ownerId di setiap collection, (c) rate limiting via request.time diff.
  • P0MAuth brute force + rate limit
    Enable Firebase Auth App Check (reCAPTCHA v3 / Play Integrity). Set Cloud Functions onCall rate limit max 10/min per UID. Anonymous quota max 1000/day per IP via App Check.
  • P1MUU PDP compliance checklist
    Dokumen: (1) lawful basis per data category (consent vs legitimate interest), (2) retention period eksplisit (saat ini implicit), (3) DPO contact email di privacy-policy.html, (4) data subject request endpoint (export/delete).
  • P1SXSS audit innerHTML usage
    Grep innerHTML di public/js/*.js — currently digunakan di history, encyclopedia, admin. Wrap dengan DOMPurify.sanitize() atau switch ke textContent/createElement.
  • P1MData export & deletion endpoint
    Cloud Function requestDataExport → ZIP semua Firestore docs per UID, kirim signed URL via Auth email. requestAccountDeletion → 30-day grace period, full purge termasuk Auth user + storage.
  • P2LPenetration test eksternal
    Engage 1 vendor (HackerOne / local Indonesian cybersec firm) untuk web pentest pre-launch Pro tier. Scope: auth bypass, Firestore rules bypass, CSP bypass, IDOR di Firestore docs.

! ⚠️ Realistic vs Stretch Flags

Tidak semua action item akan tercapai dalam timeline. Berikut transparency:

Realistic — high confidence (90%+)

  • Cat 1, 3, 4 (Performance / A11y / SEO) — eksekusi pure technical, founder-controllable
  • Cat 2 partial (CSP + Firestore rules) — eksekusi tanpa external dependency
  • Cat 9 partial (analytics, streak, push) — bisa shipped tanpa partnership
  • Cat 12 partial (public repo, CONTRIBUTING) — pure repo hygiene

Stretch — needs external partnership / capital

  • Cat 5 OTA + multi-device — butuh firmware engineer dedicated
  • Cat 7 federated learning — butuh user scale ≥10rb dulu, butuh ML research time
  • Cat 8 IRB + RS partnership — slow burn 12+ bulan, butuh clinical co-founder atau advisor
  • Cat 8 Class IIa MDR (EU) — butuh capital ~€150k + RA consultant
  • Cat 10 Stripe global expansion — butuh entity di Singapore/EU untuk billing
  • Cat 11 expansion ASEAN — butuh local partnership tiap negara

Risks yang bisa blok timeline

  • Kemenkes klasifikasi medical device — bisa expand scope dari wellness ke SaMD, butuh QMS ISO 13485
  • BLE Web API support fragmentation — iOS Safari masih limited, butuh companion native app jangka panjang
  • Dataset access (DeepBeat Stanford, MESA NSRR) — butuh IRB approval institusional
  • Burnout founder — 87 action items × 18 bulan = ~5 items/bulan, perlu prioritize ruthlessly
