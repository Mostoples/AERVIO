5 Metodologi Audit XAI

Audit dilakukan dengan kriteria objektif per fitur. Setiap "✓" harus traceable secara visual di UI atau code.

Feature Importance (FI) ✓ jika UI menampilkan kontribusi tiap factor (weight bar, percentage)
✗ jika hanya output single value tanpa breakdown
Citation ✓ jika ada paper DOI/PMID atau guideline (WHO/AHA/GINA/dll) yang dirujuk di UI atau code comment
✗ jika tidak ada referensi peer-reviewed atau institusional
Formula ✓ jika formula matematis exposed (di ml-improvement-plan.html, ml-results-report.html, atau model field di JS)
✗ jika hanya output dari black-box fungsi tanpa documentation
Data Source ✓ jika dataset/sensor input clearly disclosed (e.g., "MAX30102 PPG", "PhysioNet AF Challenge", "BMKG ISPU")
✗ jika tidak ada disclosure tentang asal data

Threshold Level: L3 = 4/4 kriteria · L2 = 2-3/4 · L1 = 1/4 (hanya disclaimer) · L0 = 0/4
